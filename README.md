# MNIST CNN 手写数字识别画板

> 这是一个基于 PyTorch CNN + Tkinter 画板的全链路 demo，用来打通数据、训练、保存模型、部署推理和交互识别流程。

![demo](assets/demo.gif)

## 项目动机

这个项目的目标不是刷新 MNIST 上的 SOTA，而是完整走通一个深度学习小项目从数据到应用的闭环。相比只写训练脚本，我更想把 `数据加载 -> 模型训练 -> 权重保存 -> 推理加载 -> GUI 交互` 串起来，形成一个可以实际点击和测试的 demo。

MNIST 本身很基础，但它适合用来理解图像分类项目的最小完整形态：输入是什么、模型怎么学、推理时数据格式如何保持一致，以及为什么训练集效果好不代表手动画板一定稳定。

## 模型结构

输入为 `1 x 28 x 28` 的灰度图像，模型是一个轻量 CNN。

| 层级 | 配置 | 输出含义 |
| --- | --- | --- |
| Input | `1 x 28 x 28` | MNIST 灰度图 |
| Conv1 | `32 filters, 3 x 3, padding=1` | 提取低级笔画和边缘特征 |
| MaxPool | `2 x 2` | 特征图从 `28 x 28` 降到 `14 x 14` |
| Conv2 | `64 filters, 3 x 3, padding=1` | 提取更复杂的局部形状 |
| MaxPool | `2 x 2` | 特征图从 `14 x 14` 降到 `7 x 7` |
| Flatten | `64 x 7 x 7` | 展平成分类向量 |
| FC | `128` | 聚合图像特征 |
| Dropout | `0.5` | 降低过拟合风险 |
| FC | `10` | 输出数字 `0-9` 的 logits |

参数量：`421,642`。

## 训练设置

| 项目 | 设置 |
| --- | --- |
| 数据集 | `MNIST`，训练集 60,000 张，测试集 10,000 张 |
| optimizer | `Adam` |
| lr | `1e-3` |
| batch size | 训练 `64`，测试 `1000` |
| epochs | `3` |
| 设备 | 自动选择 `CUDA`，不可用时使用 `CPU` |

## 结果

测试准确率：约 `99%`，用于验证基础 CNN 在 MNIST 上的分类能力。

训练耗时：本机参考约 `1-3 分钟`，取决于 CPU / GPU 和 PyTorch 环境。

单张推理时间：本机参考为毫秒级，GUI 中主要耗时通常来自画板图像预处理和界面刷新。

## 画板交互设计

Tkinter 画板负责把用户手写输入转换成模型可以理解的 `28 x 28` 灰度张量。整体流水线如下：

```text
鼠标绘制 -> PIL 像素提取 -> 裁剪有效区域 -> 居中补边 -> resize 28 x 28 -> 反色归一化 -> CNN 推理
```

画板背景是白色、笔迹是黑色，而 MNIST 中通常是黑底亮字，所以推理前需要做反色处理。随后使用和训练阶段一致的 `Normalize((0.1307,), (0.3081,))`，尽量让输入分布靠近 MNIST。

这里最大的坑是：画板分布和 MNIST 训练分布有 gap。真实手写轨迹的粗细、位置、倾斜角度、数字大小都可能和 MNIST 不一致，所以即使测试集准确率很高，画板输入也可能出现误判。

## 我学到了什么

- `torchvision.datasets.MNIST` 可以快速完成数据下载、划分和 transform 绑定，适合搭建最小训练闭环。
- CNN 的 `Conv2d -> ReLU -> MaxPool` 能逐步从像素中提取笔画、边缘和局部形状，比直接用全连接层处理图片更适合图像任务。
- 训练和推理必须保持输入处理一致，包括灰度化、尺寸缩放、反色方向和 `Normalize` 参数，否则 GUI 里的效果会明显不稳定。
- `model.eval()` 和 `torch.no_grad()` 对推理流程很重要，前者关闭 Dropout 行为，后者减少不必要的梯度计算。

## 不足之处

- MNIST 过于简单，目前没有尝试 `EMNIST`、`KMNIST` 等更复杂或分布不同的数据集，泛化能力验证不够。
- 画板分布和 MNIST 训练分布存在 gap，目前没有做 domain adaptation，也没有把画板生成的数据加入训练增强。
- 未做量化压缩，模型虽然很小，但还没有尝试 `torch.quantization`、ONNX Runtime 或移动端部署优化。
- 仅支持单字符识别，不能处理多数字序列、连笔输入或自动分割多个数字。
- 模型结构在 `mnist_cnn.py`、`predict.py`、`draw.py` 中重复定义，后续维护时容易出现参数不一致。
- 画板预处理规则仍然偏手工，例如裁剪、补边和居中策略没有系统评估，对书写风格比较敏感。

## 后续方向

- 将模型结构抽到独立文件，例如 `model.py`，让训练、图片预测和 GUI 共用同一个 CNN 定义。
- 加入数据增强或收集画板样本，减少手写画板输入和 MNIST 训练集之间的分布差异。
- 尝试导出 ONNX 或量化模型，比较原始 PyTorch 推理和轻量化部署方案的差异。
